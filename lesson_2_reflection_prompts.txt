What happens when you initialize a repository? When you initialize a repository, you create a new repository. 
Why do you need to do it? You might have a project with different versions to keep track of.



How is the staging area different from the working directory and the repository? The staging area is a limbo area between the
working directory and a commit to the repository.


What value do you think it offers? The staging area adds an extra step to changing the repository.



How can you use the staging area to make sure you have one commit per logical 
change? You can use the staging area place each 
change to the file and only commit those you are sure of.



What are some situations when branches would be helpful in keeping your history 
organized? Branches are helpful when making
different versions of the program, or when experimenting with a change.

How would branches help? Branches allow you to keep a master branch as well as make commits to other branches.



How do the diagrams help you visualize the branch structure? Diagrams provide a visual representation.



What is the result of merging two branches together? Merging two branches together combines the branches into a single branch.

Why do we represent it in 
the diagram the way we do? The way it is represented in the diagram allows you to see the commits
present from both branches.
